**Step A — Rewrite +page.svelte as the inlined context owner:**

The goal is a straight merge: take InventoryGrid.svelte's full `<script>` block and template, adapt for the route file context (replace $props with data.*), add the 3 new ctx fields, and render the same template.

**Script adaptation — replace the Props block and $props() call:**

InventoryGrid.svelte has:
```typescript
type Props = { assets: ...; keys: ...; user: ...; ... };
let { assets: initialAssets, keys: initialKeys, user, ... }: Props = $props();
```

Replace with the SvelteKit route pattern:
```typescript
import type { PageProps } from './$types';
let { data }: PageProps = $props();

// Map data fields to the same variable names used throughout the script
const initialAssets = data.assets;
const initialKeys = data.assets.length > 0 ? Object.keys(data.assets[0]) : [];
const user = data.user;
const initialLocations = data.locations ?? [];
const initialStatuses = data.statuses ?? [];
const initialConditions = data.conditions ?? [];
const initialDepartments = data.departments ?? [];
const initialView = data.initialView ?? 'default';
const searchResults = data.searchResults ?? null;
```

All other script content (controllers, $state, $derived, $effect, functions) is kept verbatim — variable names are unchanged so nothing else needs updating.

**Add the 3 new fields to the ctx literal:**

Find the `const ctx = $state<GridContext>({...})` block. Add three fields at the end of the object literal (before the closing `}`):

```typescript
    // Phase 2 additions
    filteredAssetsCount: (searchResults ?? initialAssets).length,
    virtualScroll: createVirtualScroll(),
    scrollToRow: null,
```

**Remove the duplicate virtualScroll instantiation:**

InventoryGrid.svelte has `const virtualScroll = createVirtualScroll();` as a separate local const (outside ctx). Since virtualScroll is now created inside the ctx literal, replace that line with an alias:

```typescript
const virtualScroll = ctx.virtualScroll;
```

This alias lets the rest of the script continue using `virtualScroll` without changes.

**Add filteredAssetsCount sync effect:**

After all controller instantiations and before (or alongside) other $effects, add:

```typescript
$effect(() => {
  ctx.filteredAssetsCount = filteredAssets.length;
});
```

(where `filteredAssets` is the existing local $state variable in InventoryGrid.svelte's script)

**Template — copy verbatim from InventoryGrid.svelte:**

Copy the entire template block (everything after `</script>`) from InventoryGrid.svelte into +page.svelte unchanged. The template references local script variables that still exist under the same names.

The only reference that changes: any use of `user` in the template where it was a prop. Since `user` is now a local const initialized from `data.user`, it is still called `user` — no template changes needed.

**Required imports in +page.svelte:**

The import block from InventoryGrid.svelte is kept verbatim. Add only `import type { PageProps } from './$types';` at the top. Remove the `InventoryGrid` import (it no longer exists).
