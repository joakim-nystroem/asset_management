```typescript
onclick={() => {
  uiCtx.contextMenu?.close();
  edit.startEdit(uiCtx.contextMenu!.row, uiCtx.contextMenu!.col, colCtx.keys[uiCtx.contextMenu!.col], String(dataCtx.assets[uiCtx.contextMenu!.row]?.[colCtx.keys[uiCtx.contextMenu!.col]] ?? ''));
```

IMPORTANT: Close MUST happen before startEdit because startEdit sets editCtx.isEditing = true, and the contextMenu row/col references need to still be valid at that moment. Actually since `uiCtx.contextMenu!.row` is read before close() fires (JavaScript evaluates arguments before executing), the order is: capture the row/col values in the startEdit arguments, then close sets visible=false. But to be safe, capture values first:
```typescript
onclick={() => {
  const { row, col } = uiCtx.contextMenu!;
  const key = colCtx.keys[col];
  const value = String(dataCtx.assets[row]?.[key] ?? '');
  uiCtx.contextMenu?.close();
  edit.startEdit(row, col, key, value);
}}
```

**GridOverlays.svelte** — Pass onSave to FloatingEditor:

3. On line 330, change `<FloatingEditor />` to:
```svelte
<FloatingEditor onSave={(change) => {
  history.record(change.id, change.key, change.oldValue, change.newValue);
  changes.update(change);
}} />
```

This wires the save event into both:
- `history.record()` — so Ctrl+Z/Ctrl+Y can undo/redo the edit
- `changes.update()` — so the cell is marked dirty and changeCtx.hasUnsavedChanges becomes true

Note: `changes.update()` may already be called elsewhere for edits, but currently it is NOT — the only place `changes.update()` is called is in the paste handler (GridOverlays callbacks.onPaste) and undo/redo handlers. So this is the missing link that also ensures edited cells get dirty-cell overlays.
