**DataController.svelte** — Fix URL params being wiped on view change:

```typescript
function updateSearchUrl(params: { q?: string; filters?: Filter[]; view?: string }) {
  const current = getCurrentUrlState();
  const q = params.q !== undefined ? params.q : current.q;
  const filters = params.filters !== undefined ? params.filters : current.filters;
  const view = params.view !== undefined ? params.view : current.view;

  reactiveUrl.searchParams.delete('q');
  reactiveUrl.searchParams.delete('filter');
  reactiveUrl.searchParams.delete('view');
  if (q) reactiveUrl.searchParams.set('q', q);
  if (filters && filters.length > 0) {
    filters.forEach(f => reactiveUrl.searchParams.append('filter', `${f.key}:${f.value}`));
  }
  reactiveUrl.searchParams.set('view', view || 'default');
  replaceState(new URL(reactiveUrl), {});
}
```

This way `handleViewChange({ view: viewName })` no longer wipes `q` and `filters` — they fall through from `getCurrentUrlState()`.

The problem: GridContextProvider initializes `dataCtx.assets = []`. DataController populates it via `$effect(() => { dataCtx.assets = assets; })` which is deferred until after the first render. GridContainer checks `{#if assets.length > 0}` and sees empty array on first paint, showing "Query successful, but no data" for one frame.

**Fix approach:** Move the initial data seeding from `$effect` to synchronous assignment in DataController's script block.

In **DataController.svelte**:

1. Lines 64-67 currently use `$effect` to sync data into contexts:
```typescript
$effect(() => { dataCtx.assets = assets; });
$effect(() => { dataCtx.baseAssets = baseAssets; });
$effect(() => { dataCtx.filteredAssetsCount = filteredAssets.length; });
$effect(() => { dataCtx.user = data.user ?? null; });
```

2. Add synchronous seed assignments BEFORE (above) the `$effect` blocks. These run during component initialization (script block execution), before the first render:
```typescript
// Synchronous seed — prevents "no data" flash on first render
dataCtx.assets = [...(data.searchResults ?? data.assets ?? []), ...rowGen.newRows];
dataCtx.baseAssets = data.assets ?? [];
dataCtx.filteredAssetsCount = (data.searchResults ?? data.assets ?? []).length;
dataCtx.user = data.user ?? null;
columnCtx.keys = (data.assets?.[0] ? Object.keys(data.assets[0]) : []);
```

3. Keep the existing `$effect` blocks as they are — they handle subsequent reactive updates (when filteredAssets changes due to search, etc.). The synchronous seed only covers the initial render frame.

This ensures GridContainer sees populated `dataCtx.assets` on the very first render, eliminating the "no data" flash.
