Create `frontend/src/lib/components/grid/DataController.svelte` as a renderless component (no template markup, only `<script>`).

**DataController receives one prop:** `data` — the route data from `+page.server.ts` (contains initialAssets, user, locations, statuses, validationConstraints, etc.).

**DataController reads these contexts:**
- `getDataContext()` — reads/writes assets, baseAssets, filteredAssetsCount, user
- `getChangeContext()` — reads/writes hasUnsavedChanges, hasInvalidChanges
- `getValidationContext()` — reads validationConstraints
- `getViewContext()` — reads/writes activeView, virtualScroll, scrollToRow
- `getSortContext()` — reads/writes sortKey, sortDirection
- `getColumnContext()` — reads keys
- `getSelectionContext()` — reads/writes dirtyCells
- `getEditingContext()` — reads isEditing (for URL effect guard)
- `getUiContext()` — writes headerMenu, contextMenu instances

**Logic to extract from +page.svelte into DataController:**

1. **Initial data setup:** On mount, populate dataContext with `data.assets`, `data.user`, etc. Set `validationCtx.validationConstraints` from data. Set initial `columnCtx.keys`.

2. **URL search $effect:** The large effect that reads `SvelteURL.searchParams` and calls searchManager to filter/sort assets. This entire block moves here.

3. **commitChanges():** POST changed cells to `/api/update`, clear dirtyCells, update changeContext.

4. **discardChanges():** Restore assets from baseAssets, clear dirtyCells and changes.

5. **addNewRows():** Generate empty rows, append to dataContext.assets.

6. **applySort():** Sort the assets array by key/direction, write to sortContext.

7. **handleFilterSelect():** Update URL searchParams with filter, trigger search effect.

8. **handleViewChange():** Switch activeView, update keys for the view, update URL.

9. **handleRealtimeUpdate():** Patch local assets from WebSocket delta messages.

**Action callbacks pattern:** DataController needs to expose commit/discard/addRows/viewChange to Toolbar. Since Toolbar reads context (not DataController imports), write action callbacks into `dataContext`:
```typescript
const dataCtx = getDataContext();
dataCtx.commit = commitChanges;
dataCtx.discard = discardChanges;
dataCtx.addRows = addNewRows;
```
Add these function fields to the `DataContext` type in gridContext.svelte.ts:
```typescript
export type DataContext = {
  assets: Record<string, any>[];
  baseAssets: Record<string, any>[];
  filteredAssetsCount: number;
  user: SafeUser | null;
  // Action callbacks (set by DataController)
  commit?: () => Promise<void>;
  discard?: () => void;
  addRows?: (count: number) => void;
  navigateError?: (direction: 'next' | 'prev') => void;
};
```

**In +page.svelte:** Remove the extracted logic (URL effect, commitChanges, discardChanges, addNewRows, sort/filter handlers, realtimeUpdate). Keep ONLY the context setters and component rendering. Add `<DataController {data} />` as a child. Do NOT yet do the full rewrite to < 60 lines — that is Plan 05. Just remove the logic that moved to DataController and verify the app still works.

Also remove the `pageActions` assignment from +page.svelte and remove the `ActionsContext` type/pair from gridContext.svelte.ts (since all pageActions consumers were already migrated in Plan 03). Remove `applySort` and `handleFilterSelect` callback fields too — DataController owns those directly.
