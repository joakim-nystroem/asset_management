Create a new directory `frontend/src/lib/context/` and the file `gridContext.svelte.ts`.

This file defines the unified GridContext type and exports the [getter, setter] pair via
`createContext<GridContext>()` from `svelte`.

The GridContext type must cover all cross-component state from the current singletons:

```typescript
import { createContext } from 'svelte';
import { SvelteMap } from 'svelte/reactivity';

export type GridCell = { row: number; col: number };

export type ValidationConstraints = Record<string, string[]>;

export type GridContext = {
  // Edit state (from editManager)
  isEditing: boolean;
  editKey: string | null;           // which column key is being edited
  editRow: number;                  // -1 when not editing
  editCol: number;                  // -1 when not editing
  editOriginalValue: string;
  editOriginalColumnWidth: number;
  inputValue: string;

  // Selection state (from selectionManager)
  selectionStart: GridCell;
  selectionEnd: GridCell;
  isSelecting: boolean;
  isHiddenAfterCopy: boolean;
  copyStart: GridCell;
  copyEnd: GridCell;
  isCopyVisible: boolean;
  dirtyCells: Set<string>;          // "row,col" strings

  // Change / validation state (from changeManager + validationManager)
  hasUnsavedChanges: boolean;
  hasInvalidChanges: boolean;
  validationConstraints: ValidationConstraints;

  // Column/row geometry (from columnManager + rowManager)
  columnWidths: SvelteMap<string, number>;
  rowHeights: SvelteMap<number, number>;  // rowIndex -> height
  resizingColumn: string | null;

  // Sort state (from sortManager)
  sortKey: string | null;
  sortDirection: 'asc' | 'desc' | null;

  // View state (from viewManager)
  activeView: string;

  // Column keys (from +page.svelte)
  keys: string[];
};

export const [getGridContext, setGridContext] = createContext<GridContext>();
```

Important constraints:
- Use `SvelteMap` (from `svelte/reactivity`) for `columnWidths` and `rowHeights` — plain `Map`
  mutations are NOT reactive via Svelte 5 deep proxy (see research pitfall 3)
- Use `createContext<GridContext>()` with the type parameter — without it, getter returns `unknown`
  and svelte-check fails (see research pitfall 5)
- `dirtyCells` is a plain `Set<string>` inside `$state` — Svelte 5 deep proxy handles Set reactivity
  for assignment; for granular `.add()/.delete()` reactivity use `SvelteSet` if needed (use plain
  Set initially, swap to SvelteSet if reactivity issues arise)
- `createContext()` is called at module level (not inside a function) — this is correct and safe;
  it creates the key object, not the context value

**Part A: Create InventoryGrid.svelte**

Create `frontend/src/lib/components/grid/InventoryGrid.svelte`. This component:
1. Receives data props from +page.svelte (assets, keys, user, metadata for constraints, etc.)
2. Calls `setGridContext(ctx)` synchronously in `<script>` — this is the context provider
3. Owns the grid template HTML that currently lives in +page.svelte
4. All the existing singleton imports in InventoryGrid are OK for now — they will be replaced in
   Plans 02-06. The goal here is to create the component and set the context.

The `<script>` section structure:
```svelte
<script lang="ts">
  import { setGridContext } from '$lib/context/gridContext.svelte.ts';
  import { SvelteMap } from 'svelte/reactivity';
  import type { GridContext } from '$lib/context/gridContext.svelte.ts';
  import type { SafeUser } from '$lib/types';
  // ... import all the existing singletons that the template currently needs

  type Props = {
    assets: Record<string, any>[];
    keys: string[];
    user: SafeUser | null;
    // Include any other props currently used in the grid template from +page.svelte data
    // (constraints, metadata for dropdowns, etc.)
  };

  let { assets, keys, user, ...otherProps }: Props = $props();

  // Create the mutable $state context object
  // IMPORTANT: setGridContext must be called before any $effect or await
  const ctx = $state<GridContext>({
    isEditing: false,
    editKey: null,
    editRow: -1,
    editCol: -1,
    editOriginalValue: '',
    editOriginalColumnWidth: 0,
    inputValue: '',
    selectionStart: { row: -1, col: -1 },
    selectionEnd: { row: -1, col: -1 },
    isSelecting: false,
    isHiddenAfterCopy: false,
    copyStart: { row: -1, col: -1 },
    copyEnd: { row: -1, col: -1 },
    isCopyVisible: false,
    dirtyCells: new Set(),
    hasUnsavedChanges: false,
    hasInvalidChanges: false,
    validationConstraints: {},
    columnWidths: new SvelteMap(),
    rowHeights: new SvelteMap(),
    resizingColumn: null,
    sortKey: null,
    sortDirection: null,
    activeView: 'default',
    keys: keys,
  });

  setGridContext(ctx);

  // ... rest of the script: existing variables, effects, handlers from +page.svelte
</script>

<!-- grid template moved from +page.svelte -->
```

Move the ENTIRE grid template markup from +page.svelte into InventoryGrid.svelte. Keep all
existing logic (virtualScroll, event handlers, etc.) intact in InventoryGrid.svelte. Existing
singleton imports continue to work in InventoryGrid for now — they will be replaced in later plans.
