<objective>
Replace DOM-crawling cell identification with per-cell button elements that know their own identity from render scope.

Purpose: Cells become interactive buttons with callback props. GridOverlays stops extracting cell identity from `data-*` attributes via `closest()` -- instead, GridRow calls back with `(row, col, value)` directly. Selection is derived from anchor points, guaranteeing complete rectangles even with fast mouse movement.

Output: GridRow renders `<button>` cells, all mouse interaction goes through callbacks, `selectedCells` is `$derived` from bounds, zero DOM crawling in cell interaction paths.
</objective>

**1. Create callback functions** that GridRow will call (these replace the DOM-crawling handlers):

```typescript
function handleCellMouseDown(row: number, col: string, e: MouseEvent) {
  if (e.button !== 0) return; // left click only for selection
  setOpenPanel('contextMenu'); // close other panels
  if (editingCtx.isEditing) {
    selectCell(row, col);
    return;
  }
  startSelection(row, col, e);
}

function handleCellMouseEnter(row: number, col: string) {
  if (resizeDrag) return;
  if (!editingCtx.isEditing) {
    extendSelection(row, col);
  }
}

function handleCellDblClick(row: number, col: string, value: string, e: MouseEvent) {
  if (col === 'id') {
    toastState.addToast('ID column cannot be edited.', 'warning');
    return;
  }
  e.preventDefault();
  selectCell(row, col);
  startCellEdit(row, col);
}

function handleCellContextMenu(row: number, col: string, value: string, e: MouseEvent) {
  e.preventDefault();
  selectCell(row, col);
  setOpenPanel('contextMenu');
  openContextMenu(e, row, col, value, col); // col IS the key now
}
```

**4. The new `handleContainerMouseDown`** only handles non-cell concerns:
```typescript
function handleContainerMouseDown(e: MouseEvent) {
  // Resize handle check
  const handle = (e.target as HTMLElement).closest('[data-resize-handle]') as HTMLElement | null;
  if (handle) {
    setOpenPanel();
    const key = handle.dataset.resizeHandle!;
    const startWidth = colWidthCtx.widths.get(key) ?? DEFAULT_WIDTH;
    resizeDrag = { key, startX: e.clientX, startWidth };
    document.body.style.cursor = 'col-resize';
    e.preventDefault();
    e.stopPropagation();
    return;
  }
  // Header click -- let GridHeader handle
  const headerCol = (e.target as HTMLElement).closest('[data-header-col]') as HTMLElement | null;
  if (headerCol) return;
  // Everything else (cell clicks) handled by GridRow callbacks -- do nothing here
}
```

**7. Add derived selectedCells** (SEL-02):
```typescript
const selectedCells = $derived.by(() => {
  const start = selCtx.selectionStart;
  const end = selCtx.selectionEnd;
  if (start.row === -1 || start.col === '') return [];

  const startIdx = assets.findIndex((a: Record<string, any>) => a.id === start.row);
  const endIdx = assets.findIndex((a: Record<string, any>) => a.id === end.row);
  if (startIdx === -1 || endIdx === -1) return [];

  const minRow = Math.min(startIdx, endIdx);
  const maxRow = Math.max(startIdx, endIdx);
  const startColIdx = keys.indexOf(start.col);
  const endColIdx = keys.indexOf(end.col);
  if (startColIdx === -1 || endColIdx === -1) return [];
  const minCol = Math.min(startColIdx, endColIdx);
  const maxCol = Math.max(startColIdx, endColIdx);

  const cells: { row: number; col: string }[] = [];
  for (let r = minRow; r <= maxRow; r++) {
    for (let c = minCol; c <= maxCol; c++) {
      cells.push({ row: assets[r].id, col: keys[c] });
    }
  }
  return cells;
});
```
This is not consumed by rendering yet (Phase 2 keyboard handler will use it), but it establishes the derived array for verification.
