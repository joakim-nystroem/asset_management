8. **URL-driven search $effect** (lines 192-287) — THIS IS THE KEY MIGRATION. Instead of inline async fetch, this effect now enqueues events:
   - Keep the `skipInitialFetch` guard (lines 192, 209-212)
   - Keep the searchManager sync (lines 200-207 inside untrack)
   - After skipInitialFetch check, determine what changed:
     - If `urlView !== currentView` (where currentView = `untrack(() => viewCtx.activeView)`): enqueue `{ type: 'VIEW_CHANGE', view: urlView, q, filters }`
     - Else if `q || filters.length > 0`: enqueue `{ type: 'FILTER', q, filters, view: urlView }`
     - Else (no q, no filters, same view = clear filter): enqueue `{ type: 'FILTER', q: '', filters: [], view: urlView }` — this triggers the "reset to baseAssets" path in the handler
   - Remove the `cancelled` flag pattern — no longer needed since the queue serializes (only one handler runs at a time). The `return () => { cancelled = true }` cleanup is unnecessary.

10. **Action callback wiring** (lines 580-587) — THIS CHANGES. Instead of registering direct functions, wire through the queue:
    ```typescript
    $effect(() => {
      dataCtx.commit = () => queue.enqueue({ type: 'COMMIT', mode: 'update' });
      dataCtx.discard = () => queue.enqueue({ type: 'DISCARD' });
      dataCtx.addRows = () => queue.enqueue({ type: 'COMMIT', mode: 'create' });
      dataCtx.addNewRow = handleAddNewRow;       // local-only, NOT queued
      dataCtx.navigateError = navigateToError;    // local-only, NOT queued
      dataCtx.viewChange = handleViewChange;      // calls updateSearchUrl → triggers URL $effect → queue
    });
    ```
