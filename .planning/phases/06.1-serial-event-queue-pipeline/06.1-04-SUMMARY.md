## Accomplishments

- Removed stale guard (viewName === viewCtx.activeView) that silently dropped return-to-current-view clicks

- Browser back/forward still works via unchanged URL $effect (popstate -> reactiveUrl -> $effect -> enqueue)

## Decisions Made
- handleViewChange enqueues directly instead of through URL/$effect — Svelte 5 batches effects to next microtask, coalescing intermediate URL values and dropping rapid view switches
- URL $effect kept unchanged — still needed for browser back/forward navigation (popstate updates page.url which syncs to reactiveUrl)
- updateSearchUrl passed via dependency injection to EventHandler — maintains the existing DI pattern, keeps EventHandler testable
