## What Was Built

### EventQueue.svelte.ts
- `GridEvent` discriminated union with 5 variants:
  - `CommitEvent { type: 'COMMIT'; mode: 'update' | 'create' }` — update subsumes add-rows via mode field
  - `FilterEvent { type: 'FILTER'; q, filters, view }`
  - `DiscardEvent { type: 'DISCARD' }`
  - `ViewChangeEvent { type: 'VIEW_CHANGE'; view, q, filters }` — carries q/filters for post-view-load re-application
  - `WsDeltaEvent { type: 'WS_DELTA'; payload: { id, key, value } }`
- `createEventQueue(handler)` — Promise-chain serial enforcer: `chain = chain.then(async () => { ... })`
  - `$state(false)` isProcessing reactive flag
  - Debug: `console.log('[EventQueue] START/DONE:', event.type)` + 500ms artificial delay
  - Error-safe: all handler errors caught, logged, queue continues
- `EventQueueInstance` return type exported

### EventHandler.svelte.ts
- `createEventHandler(deps)` — dependency injection factory
  - Takes getter/setter pairs for `baseAssets`/`filteredAssets` (preserves Svelte 5 reactivity)
  - Takes all required context refs and controller instances
- Exhaustive switch dispatch `handle(event: GridEvent)` covering all 5 types
- Handler implementations faithfully extracted from `DataController.svelte`:
  - `handleCommitUpdate()` — POST `/api/update`, clear changes, resetAll selection, toast
  - `handleCommitCreate()` — validate newRows, POST `/api/create/asset`, append createdRows, clearNewRows, toast
  - `handleFilter()` — GET `/api/assets?q=...&filter=...`, revert pending changes if any, set filteredAssets
  - `handleViewChange()` — validate view, GET `/api/assets?view=`, falls through to handleFilter if q/filters present
  - `handleDiscard()` — revert filteredAssets in-place, clearCommitted history, clear changes/newRows, toast
  - `handleWsDelta()` — updateAssetInList on both filteredAssets and baseAssets
- `updateAssetInList` helper copied from DataController — in-place mutation by id
