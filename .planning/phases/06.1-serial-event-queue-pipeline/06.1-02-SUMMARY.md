## What Was Built

### EventListener.svelte

A renderless Svelte 5 component (no template markup) that replaces DataController in the component tree. Receives `data: PageProps['data']` as its sole prop.

**Architecture:**
- Reads all 9 domain contexts + 2 controller contexts (same as DataController)
- Creates `selection`, `rowGen`, `validation` controllers at sync init
- Creates `createEventHandler` + `createEventQueue` at sync script level (not inside `$effect`)
- Owns local `$state` for `baseAssets` and `filteredAssets`

**Queue-eligible operations (serialized through FIFO queue):**
- `dataCtx.commit` → `queue.enqueue({ type: 'COMMIT', mode: 'update' })`
- `dataCtx.discard` → `queue.enqueue({ type: 'DISCARD' })`
- `dataCtx.addRows` → `queue.enqueue({ type: 'COMMIT', mode: 'create' })`
- URL $effect: view change → `queue.enqueue({ type: 'VIEW_CHANGE', ... })`
- URL $effect: search/filter → `queue.enqueue({ type: 'FILTER', ... })`
- WS delta → `queue.enqueue({ type: 'WS_DELTA', payload })`

**Local-only operations (instant, not queued):**
- `addNewRow` — pure in-memory row template + scroll
- `navigateToError` — reads invalid cell positions, calls `selection.selectCell()`
- `applySort` — in-memory sort, no network
- `handleFilterSelect` — updates URL → triggers URL $effect → queue
- `handleViewChange` — updates URL → triggers URL $effect → queue

**Key migration differences from DataController:**
- Removed `cancelled = true` cleanup — queue handles serialization
- URL $effect dispatches to queue instead of inline `(async () => {...})()`
- Action callbacks are `async` arrow wrappers (for `Promise<void>` type compatibility)
- Realtime handler goes through `queue.enqueue` instead of direct callback

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Fixed type mismatch on action callback wrappers**
- **Found during:** Task 1 verification (svelte-check)
- **Issue:** `DataContext` types `commit`, `addRows`, `discard` as `() => Promise<void>`, but `queue.enqueue()` returns `void`. Arrow functions like `() => queue.enqueue(...)` return `void`, causing TS error: "Type 'void' is not assignable to type 'Promise<void>'"
- **Fix:** Changed to `async () => queue.enqueue(...)` — `async` functions always return a `Promise`, so the assignment satisfies the type contract. The enqueue call is still synchronous; the async wrapper just wraps the void return in `Promise<void>`.
- **Files modified:** `frontend/src/lib/grid/eventQueue/EventListener.svelte`
- **Commit:** 8d815b8 (included in Task 1 commit after fix)
