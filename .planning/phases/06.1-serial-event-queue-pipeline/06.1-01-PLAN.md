This file contains two things:

**1. GridEvent discriminated union type** — 5 event types per CONTEXT.md (COMMIT subsumes ADD_ROWS, so no separate ADD_ROWS event):

```typescript
type CommitEvent = { type: 'COMMIT'; mode: 'update' | 'create' };
// mode:'update' = existing row changes (POST /api/update)
// mode:'create' = new row creation (POST /api/create/asset)
// This is how COMMIT subsumes ADD_ROWS — one event type, two modes

type FilterEvent = { type: 'FILTER'; q: string; filters: Filter[]; view: string };
type DiscardEvent = { type: 'DISCARD' };
type ViewChangeEvent = { type: 'VIEW_CHANGE'; view: string; q: string; filters: Filter[] };
// VIEW_CHANGE carries q and filters because after loading the new view's base data,
// it may need to re-apply search/filter if URL params still have q/filter values

type WsDeltaEvent = { type: 'WS_DELTA'; payload: { id: number; key: string; value: any } };

export type GridEvent = CommitEvent | FilterEvent | DiscardEvent | ViewChangeEvent | WsDeltaEvent;
```

Import `Filter` from `$lib/data/searchManager.svelte`.

**2. createEventQueue factory function** — Promise-chain serial enforcer:

- Uses `let chain: Promise<void> = Promise.resolve()` pattern
- `enqueue(event: GridEvent): void` appends to the chain via `chain = chain.then(async () => { ... })`
- Inside the `.then()`: set `isProcessing = true`, call `await handler(event)`, catch errors with `console.error('[EventQueue] Unhandled error:', err)`, set `isProcessing = false` in `finally`
- `isProcessing` is a `$state(false)` boolean — reactive for optional UI feedback
- Add debug instrumentation: `console.log('[EventQueue] START:', event.type)` before handler call, `console.log('[EventQueue] DONE:', event.type)` after, and `await new Promise(r => setTimeout(r, 500))` artificial delay BEFORE the handler call so serialization is visually confirmable in devtools
- Export the return type as `EventQueueInstance`

The factory takes a `handler: (event: GridEvent) => Promise<void>` parameter — this will be the EventHandler's dispatch function.

Return object: `{ enqueue, get isProcessing() { return isProcessing } }`

Create `EventHandler.svelte.ts` in `frontend/src/lib/grid/eventQueue/`.

This file exports a `createEventHandler` factory function. The factory receives all the context and controller references it needs as parameters (NOT via getters — it is a `.svelte.ts` file, not a `.svelte` component, so `createContext` getters are not available). The caller (EventListener.svelte) will pass these in.

**Factory parameters** (destructured object):
```typescript
createEventHandler(deps: {
  // Mutable state refs — these are reactive $state objects from the caller
  getBaseAssets: () => Record<string, any>[];
  setBaseAssets: (v: Record<string, any>[]) => void;
  getFilteredAssets: () => Record<string, any>[];
  setFilteredAssets: (v: Record<string, any>[]) => void;
  // Context refs
  dataCtx: DataContext;
  viewCtx: ViewContext;
  sortCtx: SortContext;
  editingCtx: EditingContext;
  changeCtx: ChangeContext;
  columnCtx: ColumnContext;
  // Controller refs
  changes: ChangeController;
  history: HistoryController;
  selection: SelectionController;
  rowGen: RowGenerationController;
})
```

Use getter/setter pairs for baseAssets and filteredAssets because these are local `$state` variables in the caller — the handler needs to read AND write them. Direct object property access would lose reactivity.

**Dispatch function** — `async function handle(event: GridEvent): Promise<void>` with exhaustive switch:
- `case 'COMMIT'`: if `event.mode === 'update'` call `handleCommitUpdate()`, if `event.mode === 'create'` call `handleCommitCreate()`
- `case 'FILTER'`: call `handleFilter(event.q, event.filters, event.view)`
- `case 'DISCARD'`: call `handleDiscard()`
- `case 'VIEW_CHANGE'`: call `handleViewChange(event.view, event.q, event.filters)`
- `case 'WS_DELTA'`: call `handleWsDelta(event.payload)`

**Handler implementations** — extract directly from DataController.svelte:

1. **handleCommitUpdate()** — Lines 444-481 of DataController (`commitChanges`). Identical logic: check user, check hasInvalidChanges, get validChanges, POST to `/api/update`, on success clear changes + resetAll selection + toast. On error, toast. All errors caught — never throw.

2. **handleCommitCreate()** — Lines 484-559 of DataController (`addNewRows`). Identical logic: check user, validateAll, check incomplete rows, POST to `/api/create/asset`, on success append createdRows to baseAssets AND filteredAssets (via setBaseAssets/setFilteredAssets), clear newRows + dirtyCells + resetAll + toast. On error, toast. Never throw.

3. **handleFilter(q, filters, view)** — Lines 248-283 of DataController (the non-view-change part of the URL $effect). If no q and no filters: set filteredAssets to [...baseAssets], clear sort, reset selection. Else: build URLSearchParams, fetch `/api/assets?${params}`, on success revert pending changes if any (same logic as DataController lines 265-273), clear history, clear newRows, set filteredAssets, reset selection, clear sort. On error, toast + console.error. Never throw.

4. **handleViewChange(view, q, filters)** — Lines 217-246 of DataController (the view-change part of the URL $effect). Validate view against `['default', 'audit', 'ped', 'galaxy', 'network']`. Set viewCtx.activeView. Fetch `/api/assets?view=${view}`. On success: set baseAssets. If no q and no filters: set filteredAssets too, reset selection, clear sort/changes/history/newRows, return. If q or filters exist, fall through to the filter logic (call handleFilter internally). On error, console.error + toast. Never throw.

5. **handleDiscard()** — Lines 562-578 of DataController (`discardChanges`). Identical logic: check user, revert changes in filteredAssets, clearCommitted from history, clear changes, clear newRows, clear dirtyCells, resetAll selection, toast. Synchronous but queued per CONTEXT.md locked decision.

6. **handleWsDelta(payload)** — Lines 355-368 of DataController (`handleRealtimeUpdate`). Call `updateAssetInList` on both filteredAssets and baseAssets. The `updateAssetInList` helper (lines 355-363) should be a local helper in this file.

**Important:** The `updateAssetInList` helper mutates the array in-place (finds by id, sets key/value). Copy it directly from DataController.

Import `toastState` from `$lib/components/toast/toastState.svelte`.
Import `tick` from `svelte` (used in handleAddNewRow — but handleAddNewRow stays local-only, not in handler. Actually, handleAddNewRow is NOT a queue event — it's local-only. It stays in EventListener).
