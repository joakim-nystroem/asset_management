## Resolution

root_cause: |
  TWO intertwined bugs prevent new row commits from working:

  **Bug A (Primary): Toolbar creates its own independent `rowGen` instance**
  File: `frontend/src/lib/components/grid/Toolbar.svelte` line 7
  `const rowGen = createRowGenerationController();` creates a fresh instance with zero rows.
  The EventListener's rowGen (which actually holds the new rows) is a DIFFERENT instance.
  Result: `rowGen.hasNewRows` in Toolbar is ALWAYS false, so the new-row Commit button
  (which calls `dataCtx.addRows` -> COMMIT mode:'create') is NEVER shown.

  **Bug B (Consequence): Editing new row cells activates the WRONG commit path**
  When the user edits cells in a new scaffold row, the FloatingEditor save flow records
  changes in the ChangeController (via GridOverlays onSave). This sets
  `changeCtx.hasUnsavedChanges = true`, which shows the EXISTING-row Commit button
  (calls `dataCtx.commit` -> COMMIT mode:'update'). This sends the scaffold row's fake
  temporary ID to `/api/update`, which runs UPDATE WHERE id=fakeId, matches 0 rows,
  returns success (SQL UPDATE with no matches doesn't throw), and shows a false positive
  success toast. The new row data is then cleared from local state.

  **Bug C (Secondary): Same independent-instance problem in contextMenu.svelte**
  File: `frontend/src/lib/grid/components/context-menu/contextMenu.svelte` line 14
  Creates its own rowGen instance, so Delete Row for new rows operates on wrong instance.

  **Bug D (Secondary): No required-field validation for new rows**
  File: `frontend/src/lib/grid/utils/rowGeneration.svelte.ts` lines 16-22
  `isValidValue()` allows empty strings for all fields (`if (valueStr === '') return true`).
  It only checks constraint-list membership, not whether required fields are populated.
  The proper validation with REQUIRED_FIELDS exists in `gridValidation.svelte.ts` but
  is NOT used by rowGeneration's validateAll().

fix: |
  **Fix A+B: Share rowGen instance via context (like changes/history controllers)**
  1. Add `[getRowGenControllerContext, setRowGenControllerContext]` to gridContext.svelte.ts
  2. In EventListener.svelte: publish rowGen via `setRowGenControllerContext(rowGen)` (or wire it through +page.svelte)
  3. In Toolbar.svelte: replace `createRowGenerationController()` with `getRowGenControllerContext()`
  4. In contextMenu.svelte: replace `createRowGenerationController()` with `getRowGenControllerContext()`
  5. In GridOverlays.svelte: replace `createRowGenerationController()` with `getRowGenControllerContext()`

  **Fix C: Prevent ChangeController from tracking new-row edits**
  Either:
  - (a) In FloatingEditor/GridOverlays onSave: detect if the edited row is a new row (row >= filteredAssetsCount) and call rowGen.updateNewRowField() instead of changes.update(), OR
  - (b) In EventListener, add logic to detect when both hasNewRows and hasUnsavedChanges are true and route to the correct commit path

  **Fix D: Use gridValidation's isValidValue in rowGeneration's validateAll()**
  Replace rowGeneration's simple isValidValue with a call to the ValidationController's
  isValidValue (which includes REQUIRED_FIELDS checking).
